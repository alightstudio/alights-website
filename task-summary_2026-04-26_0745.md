# 主页作品集数据源修复

## 目标
修复 alights.cn 主页「代表作品」区块的数据源：从 stash 文件（别人的灵感作品）改为作品集数据（自己的作品）。

## 关键理解
- **作品集（/works 页 + 主页代表作品）** = 自己的作品 → 储存在 siteConfig.featuredWorks（数据库）
- **创意灵感（/gallery 页）** = 别人的灵感作品 → stash 文件（stash176~stash160），保持不变
- 之前错误地将 stash176 作为主页作品集数据源，用户明确纠正：所有 stash 文件都是灵感作品

## 改动
- `src/app/page.tsx`
  - 移除 `import { stash176 } from '@/lib/stash-data'`
  - 重写 `getInitialWorks()`：从 `prisma.siteConfig.findUnique({ where: { key: 'featuredWorks' } })` 读取
  - 按 `views` 降序取前6
  - 字段映射：`image` → `coverUrl` & `thumbnail`（兼容 HomeClient 两种引用）
  - 移除不再需要的 `mapCategory()` 函数

## 结论
- 构建成功 ✅
- Vercel 部署成功 ✅（alias: alights.cn）
- /works 页和 /gallery 页未受影响（已回退之前的错误改动）
